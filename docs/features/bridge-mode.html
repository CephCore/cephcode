Bridge Mode — Remote Control & Collaboration

Control your Claude Code terminal from anywhere — via web, mobile, or IDE

What is Bridge Mode?

Bridge Mode turns your local Claude Code terminal into a remotely controllable execution environment. You can connect to it from a web browser, mobile device, or IDE extension — the AI runs on your machine, but the interface is wherever you are.

Think of it as: "Your terminal is the engine, the web is the steering wheel."

How It Works

1. Your CLI registers as a "bridge environment"

The local Claude Code registers itself with the remote control server, becoming discoverable as an available execution environment.

2. Remote clients connect via session URL

A unique session URL is generated. Open it in any browser, scan the QR code with your phone, or connect via an IDE extension.

3. Long-polling for work

Your terminal continuously polls for remote tasks. When a remote client sends a message, the bridge picks it up and executes it locally.

4. Bidirectional WebSocket

Once connected, a WebSocket link enables two-way messaging — you see the AI's output on the web, and the web sends your input back to the terminal.

Quick Start

Step 1: Authenticate

Make sure you're logged in with your claude.ai account:

claude auth login

Step 2: Start Bridge / Remote Control

Inside an active session, run the command:

/remote-control

Or from the command line with a custom name:

/remote-control my-workspace

If you're using a bridge-enabled build and the GrowthBook gate is active, the bridge will connect and you'll see a session URL.

Step 3: Connect via Web

Once connected, the terminal will show a dialog with:

  • A session URL — open it in any browser
  • A QR code — scan it with your phone for mobile access
  • Options to disconnect or continue

Opening the session URL in a browser gives you a web chat interface. The AI still runs on your local machine — you just interact through the browser.

Step 4: Disconnect

Run /remote-control again while connected to see the disconnect dialog, or use:

/remote-control
# Then select "Disconnect this session"

Use Cases

Remote Coding from Mobile

Start a long-running task on your desktop, then go mobile. Open the session URL on your phone or scan the QR code to monitor progress, ask follow-ups, or give new instructions.

Pair Programming / Review

Share the session URL with a colleague. They can see what the AI is doing in real-time and participate in the conversation without needing CLI access.

IDE Integration

Use an IDE extension as a remote frontend. The AI runs on your powerful dev machine or server, while you interact through your editor.

Headless / Server Mode

Run Claude Code on a remote server or CI environment and control it from your local browser. Useful for reviewing build logs, debugging production issues, or running automated workflows.

Key Technical Details

  • Dual implementation: v1 (env-based, replBridge.ts) and v2 (env-less, remoteBridgeCore.ts) — v2 is the newer, more secure version
  • Long-polling: Uses HTTP GET with 10s timeout instead of persistent WebSocket for work discovery
  • OAuth refresh: Built-in withOAuthRetry handles token expiration automatically
  • Security: Path traversal protection on all bridge IDs, trusted device tokens (v2), BridgeFatalError for non-retryable errors
  • Heartbeat: Periodic heartbeat extends the work lease so the server knows the session is still active
  • Permissions: Remote permission requests are sent to the control surface (web) for approval

Bridge Mode — 远程控制与协作

从任何地方控制你的 Claude Code 终端 — 网页、手机或 IDE

什么是 Bridge Mode？

Bridge Mode 将你的本地 Claude Code 终端转变为一个可远程控制的执行环境。你可以从网页浏览器、移动设备或 IDE 扩展连接它——AI 在你的机器上运行，但界面跟着你走。

可以理解为："你的终端是引擎，网页是方向盘。"

工作流程

1. CLI 注册为 "bridge 环境"

本地 Claude Code 向远程控制服务器注册自己，成为可被发现的执行环境。

2. 远程客户端通过会话 URL 连接

生成一个唯一的会话 URL。在浏览器中打开它，用手机扫描二维码，或通过 IDE 扩展连接。

3. 长轮询等待任务

你的终端持续轮询远程任务。当远程客户端发送消息时，bridge 接收它并在本地执行。

4. 双向 WebSocket

连接后，WebSocket 链路实现双向通信——你在网页上看到 AI 的输出，网页将输入发送回终端。

条件说明
Bridge Mode 构建必须使用启用了 feature('BRIDGE_MODE') 的构建版本

快速开始

第一步：登录认证

确保你已用 claude.ai 账户登录：

claude auth login

第二步：启动 Bridge

在活跃会话中运行命令：

/remote-control

或指定自定义名称：

/remote-control my-workspace

如果使用启用了 bridge 的构建且 GrowthBook 门控已激活，bridge 将连接并显示会话 URL。

第三步：通过网页连接

连接后，终端会显示一个对话框，包含：

  • 会话 URL — 在任何浏览器中打开
  • 二维码 — 用手机扫描进行移动访问
  • 选项 — 断开连接或继续

在浏览器中打开会话 URL 即可获得网页聊天界面。AI 仍在你的本地机器上运行——你只是通过浏览器与之交互。

第四步：断开连接

在已连接状态下再次运行 /remote-control 即可看到断开连接的对话框：

/remote-control
# 选择 "Disconnect this session"

使用场景

移动端远程编程

在桌面端启动一个长期运行的任务，然后出门。在手机上打开会话 URL 或扫描二维码，监控进度、追问问题或发出新指令。

结对编程 / 代码审查

将会话 URL 分享给同事。他们可以实时看到 AI 在做什么，参与对话，无需 CLI 访问权限。

IDE 集成

使用 IDE 扩展作为远程前端。AI 在你的强大开发机或服务器上运行，而你通过编辑器进行交互。

无头 / 服务器模式

在远程服务器或 CI 环境上运行 Claude Code，从本地浏览器控制它。适用于查看构建日志、调试生产问题或运行自动化工作流。

关键技术细节

  • 双重实现：v1（基于环境变量，replBridge.ts）和 v2（无需环境变量，remoteBridgeCore.ts）——v2 是更新的更安全版本
  • 长轮询：使用 HTTP GET + 10s 超时，而非持久 WebSocket 进行任务发现
  • OAuth 刷新：内置 withOAuthRetry 自动处理令牌过期
  • 安全性：所有 bridge ID 的路径穿越防护，可信设备令牌（v2），不可重试错误的 BridgeFatalError
  • 心跳：定期发送心跳延长工作租约，让服务器知道会话仍活跃
  • 权限：远程权限请求发送到控制面（网页）进行审批

Bridge Mode — ควบคุมและทำงานร่วมกันจากระยะไกล

ควบคุม Claude Code terminal ของคุณจากที่ไหนก็ได้ — ผ่านเว็บ มือถือ หรือ IDE

Bridge Mode คืออะไร?

Bridge Mode เปลี่ยน Claude Code terminal ในเครื่องของคุณให้เป็น สภาพแวดล้อมการทำงานที่ควบคุมจากระยะไกล คุณสามารถเชื่อมต่อจากเว็บเบราว์เซอร์ อุปกรณ์มือถือ หรือ IDE extension — AI รันบนเครื่องของคุณ แต่อินเทอร์เฟซอยู่ที่คุณ

เข้าใจง่าย ๆ คือ: "terminal ของคุณคือเครื่องยนต์ เว็บคือพวงมาลัย"

ขั้นตอนการทำงาน

1. CLI ของคุณลงทะเบียนเป็น "bridge environment"

Claude Code ในเครื่องลงทะเบียนตัวเองกับเซิร์ฟเวอร์ remote control กลายเป็นสภาพแวดล้อมการทำงานที่สามารถค้นหาได้

2. ไคลเอนต์ระยะไกลเชื่อมต่อผ่าน URL เซสชัน

สร้าง URL เซสชันเฉพาะ เปิดในเบราว์เซอร์ สแกน QR code ด้วยมือถือ หรือเชื่อมต่อผ่าน IDE extension

3. Long-polling เพื่อรับงาน

Terminal ของคุณจะคอยสอบถามงานระยะไกลอย่างต่อเนื่อง เมื่อไคลเอนต์ระยะไกลส่งข้อความ bridge รับและดำเนินการในเครื่อง

4. WebSocket แบบสองทิศทาง

เมื่อเชื่อมต่อแล้ว ลิงก์ WebSocket ช่วยให้สื่อสารสองทาง — คุณเห็นเอาต์พุตของ AI บนเว็บ และเว็บส่งอินพุตของคุณกลับไปยัง terminal

เริ่มต้นใช้งาน

ขั้นตอนที่ 1: เข้าสู่ระบบ

ตรวจสอบว่าคุณได้เข้าสู่ระบบด้วยบัญชี claude.ai แล้ว:

claude auth login

ขั้นตอนที่ 2: เริ่ม Bridge

ภายในเซสชันที่ใช้งานอยู่ ให้รันคำสั่ง:

/remote-control

หรือระบุชื่อที่กำหนดเอง:

/remote-control my-workspace

ถ้าคุณใช้ build ที่เปิด bridge ไว้และ GrowthBook gate ทำงาน bridge จะเชื่อมต่อและแสดง URL เซสชัน

ขั้นตอนที่ 3: เชื่อมต่อผ่านเว็บ

เมื่อเชื่อมต่อแล้ว terminal จะแสดงไดอะล็อกพร้อม:

  • Session URL — เปิดในเบราว์เซอร์ใดก็ได้
  • QR code — สแกนด้วยมือถือเพื่อเข้าถึงจากโทรศัพท์
  • ตัวเลือก — ตัดการเชื่อมต่อหรือดำเนินต่อ

การเปิด session URL ในเบราว์เซอร์จะแสดงอินเทอร์เฟซแชทบนเว็บ AI ยังคงทำงานบนเครื่องของคุณ — คุณแค่โต้ตอบผ่านเบราว์เซอร์

ขั้นตอนที่ 4: ตัดการเชื่อมต่อ

รัน /remote-control อีกครั้งขณะที่เชื่อมต่ออยู่เพื่อดูไดอะล็อกตัดการเชื่อมต่อ:

/remote-control
# จากนั้นเลือก "Disconnect this session"

กรณีการใช้งาน

เขียนโปรแกรมระยะไกลจากมือถือ

Pair Programming / รีวิวโค้ด

แชร์ session URL ให้เพื่อนร่วมงาน พวกเขาจะเห็นว่า AI กำลังทำอะไรแบบเรียลไทม์ และมีส่วนร่วมในการสนทนาโดยไม่ต้องเข้าถึง CLI

การทำงานกับ IDE

ใช้ IDE extension เป็น frontend ระยะไกล AI รันบนเครื่องพัฒนาที่ทรงพลังของคุณหรือเซิร์ฟเวอร์ ในขณะที่คุณโต้ตอบผ่าน editor

รายละเอียดทางเทคนิค

  • สองเวอร์ชัน: v1 (基于环境变量, replBridge.ts) และ v2 (ไม่ต้องใช้ env, remoteBridgeCore.ts) — v2 เป็นเวอร์ชันใหม่และปลอดภัยกว่า
  • Long-polling: ใช้ HTTP GET + 10s timeout แทน WebSocket ถาวรสำหรับการค้นพบงาน
  • OAuth refresh: withOAuthRetry ในตัว จัดการ token หมดอายุอัตโนมัติ
  • ความปลอดภัย: ป้องกัน path traversal ใน bridge ID ทั้งหมด, trusted device tokens (v2), BridgeFatalError สำหรับข้อผิดพลาดที่ไม่ควรลองใหม่
  • Heartbeat: ส่งสัญญาณ heartbeat เป็นระยะเพื่อต่ออายุงาน ให้เซิร์ฟเวอร์รู้ว่าเซสชันยังทำงานอยู่
  • สิทธิ์: คำขอสิทธิ์ระยะไกลถูกส่งไปยัง control surface (เว็บ) เพื่ออนุมัติ
